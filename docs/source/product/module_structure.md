# Module structure

The module is organized in the following folders:

- **/devops**: contains the code responsible of executing the pipeline in Azure DevOps
- **/conf**: contains the module parameters and the Databricks job configuration
- **/docs**: contains the code for this user's manual
- **/notebooks**: contains notebooks with analysis done during the project
- **/src/backstage**: contains user code
- **/tests**: contains unittests


## /conf

Contains all configuration files:

- **infra_params.json**: this file is generated automatically at the instantiation of the workspaces. It should not be edited unless you know what you are doing.
- **ml_config.json**: this file can be edited and it contains parameters to pass to aip CLI
- **project_params.json**: this file can be edited. It contains variables related to your user code


## /docs

Contains the code and product documentation of the module (this document).
The documentation can be generated by running the build.sh file located into the docs folder.
